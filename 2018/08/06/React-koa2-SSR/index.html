拥抱React16

ReactDOM.hydrate()

在React 16中，有两种方法可以在客户端进行渲染：render（）用于何时仅在客户端呈现内容，而hydrate（）用于在服务器端呈现标记之上进行渲染，如果在已经具有此服务器呈现标记的节点上调用ReactDOM.hydrate（），则React将附加事件处理程序，从而具有高性能的首次加载体验。
。 React 16降低了SSR时生成HTML的开销， 在React 15中，SSR文档中的每个HTML元素都有一个data-reactid属性，其值由递增的ID组成，文本节点被带有react-text和ID的注释包围。 在React 16中，所有ID都已从标记中删除，因此相同代码段的HTML更加精简。

关于 ReactDOMServer.renderToNodeStream() ReactDOMServer.renderToStaticNodeStream()如何返回一个Readable stream从而提升首屏的加载速度和renderToStaticMarkup() renderToStaticNodeStream() 如何去除额外的DOM属性来提升加载速度，请参考React SSR DOCS。

更进一步 🤔

  • 与react-router集成时： 由于server端没有浏览器下的全局history，需要使用StaticRouter来替代，并提供必要的context属性(request.url)。
  • readux集成时，需要在server端配置store。
  • 文件拆分，随着项目增大，将前后端共享的路由文件拆分出来是必要的。
  • 与styled-components集成时，需要 ServerStyleSheet实例下的collectStyles()将组件样式提取出来。除了满足SSR的需求外，由于styled-components是在客户端运行时动态生成样式，提前提取样式同时也提了高客户端的渲染速度。
